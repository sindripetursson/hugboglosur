Continious Integration

Continious integration server er alveg clean, þannig sjáum við skýrar dependency. Integration server er server eða virtual umhverfi sem er er sett upp í þeim eina tilgangi að vinna í ákveðnu umhverfi.

Version Control = Revision Control = Source Control

Kerfi voru með lása, minna um það í dag. Hægt að track'a breytingar aftur í tímann.

Trunk

  • The unique line of development that is not a branch(sometimes also called a Baseline or mainline)
  • Sonja segir a' þetta sé aðaldraslið, what gonna do with all that junk, all that junk inside your trunk

Branch

  • Þá er unnið á annari útgáfu af dótinu, hægt að sameina seinna.
  • Semí sandbox

Tag

  • Er svona snapshot eða eð. Getur séð hver staðan var á skrám á einhverjum tímapunkti
  • Tagg er merkt eða skýrð sérstaklega.

Allskonar annað

  • Head - nýjasta commitið
  • Revision - er ein keðja af breytingum
  • Merge - oftast eru ekki breytingar á sama stað. Conflict eru ekki hægt að automate'a
  • Revert - Lagar manns eigið sandbox aftur að þeim punkti sem seinasta uppfærlsa var gerð
  • Rollback - Hættir við commit eða bakkar um commit.. veit ekki hvort

Key advantages

  • Allir eru með sitt sandbox
  • Þetta virkar off the line
  • Hratt
  • Smíðað til að deila breitingum
  • Branching og Merging er þægilegt
  • Less management

Disadvantage

  • Það er í rauninni aldrei til seinasta útgáfa

Eininga prófanir

  • Prófa eins lítinn hluta í einu af kerfinu eins og hægt er, virkni í klasa vanalega
  • Er talin hluti af source code
  • Keyrður sem hluti af build process
  • Keyrir hratt
  • Á ábyrgð forritara
